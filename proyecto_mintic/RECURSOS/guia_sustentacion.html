2 Problema y objetivo ~1 min

"El mercado laboral colombiano tiene datos abiertos, pero están completamente fragmentados. El DANE publica los boletines de desempleo en PDF, uno por mes. La informalidad laboral está en Excel trimestrales con formatos distintos cada año. Y la demanda de formación del SENA está en una API separada.

Ningún tomador de decisiones puede cruzar esas tres fuentes manualmente cada mes. Y sin esa visión integrada, las políticas de empleo se diseñan con datos incompletos.

Nuestro objetivo fue simple: integrar todo, modelarlo con IA y hacerlo público y gratuito."
Palabras clave:fragmentadosPDF mensualExcel trimestralesAPI separadavisión integrada

3 Datos abiertos ~1 min

"Usamos cuatro fuentes, todas públicas y gratuitas.

Del DANE descargamos 33 boletines PDF con las tasas de desempleo, ocupación y participación mensual desde 2023. Del FILCO, que es el portal de MinTrabajo, descargamos más de 40 archivos Excel con datos de informalidad laboral por ciudad y sexo. Del SENA, consultamos la API de datos.gov.co con el identificador 8pqf-rmzr, que tiene los inscritos por ocupación. Y también procesamos microdatos de la GEIH, la Gran Encuesta Integrada de Hogares del DANE.

El reto no fue conseguir los datos — son abiertos. El reto fue procesarlos automáticamente, porque cada fuente tiene un formato diferente."
Si alguien pregunta "¿cómo extrajeron los PDFs?", responde: "usamos pdfplumber con una estrategia dual: primero intentamos regex en el texto, y si eso falla, extraemos directamente la tabla de las primeras 5 páginas. Funciona en el 100% de los boletines."

4 Solución e IA ~2 min

El pipeline — cómo fluyen los datos

"La solución tiene cuatro etapas. Primero, extracción: scripts de Python descargan automáticamente todos los archivos. Segundo, procesamiento: parseamos los PDFs y validamos la integridad de cada archivo. Tercero, transformación: usamos PySpark para procesar los datos en paralelo y cargarlos a una base de datos PostgreSQL con esquema estrella — siete tablas. Cuarto, el modelo de IA lee esa base de datos y genera el pronóstico."

El modelo IA — cómo explicarlo

"Para predecir el desempleo usamos Holt-Winters, también llamado suavización exponencial triple.

En términos simples: el modelo aprende tres cosas de la serie histórica — el nivel promedio actual, la tendencia de subida o bajada, y el patrón estacional que se repite cada año — porque el desempleo en Colombia sube en enero y baja en el segundo semestre, siempre.

Entrenamos el modelo con el 80% de los datos — 27 meses — y evaluamos con el 20% restante. El resultado: MAE de 0.43 puntos porcentuales. Eso significa que si la tasa real es 10.5%, el modelo predice entre 10.1% y 10.9%. Para un indicador macroeconómico mensual, eso es muy preciso."
Concepto clave:nivel + tendencia + estacionalidad80/20 train/testMAE 0.43 ppPySparkPostgreSQL 7 tablas
Si te preguntan "¿por qué Holt-Winters y no una red neuronal?", responde: "Holt-Winters es la elección correcta para una serie temporal mensual con 34 puntos. Las redes neuronales necesitan miles de datos para aprender; con 34 meses sobreajustan. Holt-Winters captura exactamente lo que la serie tiene: tendencia y estacionalidad."

❓ Preguntas probables del jurado — cómo responderlas

¿Por qué eligieron Holt-Winters y no una red neuronal o LSTM?
"Con 34 puntos de datos mensuales, una red neuronal sobreajustaría — aprendería el ruido en lugar de la señal. Holt-Winters es el modelo estadísticamente correcto para esta cantidad de datos y este tipo de serie: tiene tendencia clara y estacionalidad anual marcada, que es exactamente lo que el modelo captura. Los resultados lo confirman: MAE de 0.43 pp está bien por debajo del umbral del concurso."
¿Cómo garantizan que el modelo no está sobreajustado?
"Usamos una división estricta hold-out 80/20 en el tiempo — nunca vio los últimos 7 meses durante el entrenamiento. Las métricas que reportamos son exclusivamente sobre ese conjunto de prueba que el modelo no conocía. Además, los parámetros alpha, beta y gamma se optimizan automáticamente por el algoritmo, no los ajustamos manualmente para hacer que el resultado se vea mejor."
¿Por qué los datos del SENA son solo 2019–2020 si hay datos más recientes?
"El dataset de inscritos por ocupación en datos.gov.co con identificador 8pqf-rmzr, que es el que cumple los requisitos del concurso al estar en el portal oficial, tiene cobertura hasta 2020. Usamos ese dataset porque es el que está disponible públicamente en la plataforma de datos abiertos del gobierno."
¿El dashboard se actualiza automáticamente con datos nuevos?
"La arquitectura está diseñada para eso. Los scripts de extracción pueden correr periódicamente, el pipeline procesa los nuevos datos y el modelo se reentrana. En la versión actual el proceso es manual — se ejecuta cuando el DANE publica un nuevo boletín — pero está listo para automatizarse con un scheduler. Es la siguiente iteración natural del proyecto."
¿Qué es PySpark y para qué lo usaron?
"PySpark es el API de Python para Apache Spark, que es un motor de procesamiento de datos distribuido. Lo usamos en la etapa de transformación para procesar los 40 archivos Excel de informalidad en paralelo y cargarlos a PostgreSQL vía JDBC. Para este volumen no era estrictamente necesario, pero lo incluimos porque el concurso valora el uso de tecnologías de Big Data y porque escala sin cambios si el volumen crece."
¿Qué es un esquema estrella y por qué lo usaron?
"Un esquema estrella es un diseño de base de datos donde tenés una tabla central de hechos — los datos que mides, como la tasa de desempleo mensual — y tablas de dimensiones alrededor que dan contexto: el período, el departamento, la ciudad. Lo usamos porque facilita las consultas analíticas del dashboard, separa los datos de referencia de los datos de medición, y es el estándar en data warehousing."
¿Cuál es la precisión real del modelo para el segundo semestre de 2026?
"El MAE de 0.43 pp es sobre el conjunto de prueba histórico, donde conocemos el valor real. Para el pronóstico futuro, el error esperado crece con el horizonte — por eso reportamos un intervalo de confianza del 90%, calculado como predicción más o menos 1.645 veces el MAE. Eso nos da una banda de aproximadamente ±0.7 puntos porcentuales para el primer mes y algo mayor hacia el sexto mes."
¿Cómo garantizan la reproducibilidad?
"Tres mecanismos: Docker Compose que levanta toda la infraestructura con un comando, requirements.txt con versiones fijas de todas las dependencias Python, y GitHub Actions que corre automáticamente los tests en cada commit para asegurar que nada se rompe. Cualquier persona puede clonar el repositorio y replicar exactamente el mismo resultado."
