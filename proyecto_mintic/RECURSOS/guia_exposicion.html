02 Problema y Objetivo ~2 min
"La tasa de desocupación en Colombia oscila entre 8% y 20% con alta variación estacional. Es el indicador más crítico del mercado laboral, pero no existe ninguna herramienta pública que lo proyecte.

El problema no es falta de datos — el problema es que esos datos están completamente fragmentados: el DANE publica los boletines en 33 PDFs, uno por mes; la informalidad laboral está en más de 40 archivos Excel trimestrales del FILCO; y la demanda de formación del SENA está en una API separada.

El MinTrabajo y el SENA diseñan políticas con esa información dispersa, sin visibilidad anticipada.

Nuestro objetivo fue: integrar todo en un pipeline ETL automatizado, aplicar Holt-Winters para proyectar la desocupación a 6 meses con un intervalo de confianza del 90%, y publicar el resultado en un dashboard web de acceso libre."

La palabra clave es "fragmentado". Repítela. El jurado tiene que sentir el dolor del problema antes de ver la solución.
03 Datos Abiertos — Fuentes, Variables y Tratamiento ~1 min 30 seg
"Usamos cuatro fuentes, todas públicas y gratuitas.

Del DANE descargamos 33 boletines PDF de la GEIH — la Gran Encuesta Integrada de Hogares — con las tasas de desocupación, ocupación y participación mensual desde mayo 2023 hasta febrero 2026. Para extraer los números usamos una estrategia dual: primero regex en el texto plano y si falla, extracción directa de las tablas de las primeras 5 páginas con pdfplumber.

Del FILCO, portal de MinTrabajo, descargamos más de 40 anexos Excel trimestrales con la tasa de informalidad laboral en 13 ciudades, desagregada por sexo.

Del SENA, consultamos directamente la API de datos.gov.co — identificador 8pqf-rmzr — con los inscritos por área de ocupación.

Los tres datasets se integran en un esquema estrella de 7 tablas en PostgreSQL 15, cargadas con PySpark JDBC dentro de Docker."

04 Solución e IA — Pipeline ETL + Modelo + Dashboard ~3 min
"La solución tiene cuatro fases que siguen la metodología CRISP-ML.

Fase 1 — DataSource: scripts de Python descargan automáticamente PDFs, Excel y datos de la API.
Fase 2 — DataProcess: limpieza, validación de rangos y parseo de todos los archivos.
Fase 3 — DataTransform: PySpark procesa los datos en paralelo y los carga a PostgreSQL vía JDBC.
Fase 4 — DataProduct: el modelo de IA genera el pronóstico y el dashboard lo publica.

Sobre el modelo: usamos Holt-Winters, también llamado suavización exponencial triple. Lo elegimos porque la serie de desocupación tiene tres componentes claros: un nivel base, una tendencia bajista desde 2023, y una estacionalidad anual — el desempleo sube en enero y baja en el segundo semestre, todos los años.

Entrenamos con el 80% de los datos — 27 meses — y evaluamos en el 20% restante — 7 meses — que el modelo nunca vio. Los parámetros alfa, beta y gamma se optimizan automáticamente.

La infraestructura corre en Docker Compose con 3 contenedores: PostgreSQL, Jupyter con PySpark y Adminer. El dashboard es una aplicación Next.js desplegada en Vercel."
CRISP-ML(Q) · 4 fases Holt-Winters · nivel + tendencia + estacionalidad 80% train / 20% test PySpark JDBC Docker · 3 contenedores Next.js · Vercel
Para explicar Holt-Winters sin tecnicismos di: "el modelo aprende el nivel del desempleo, aprende si está subiendo o bajando, y aprende que cada enero sube y cada julio baja. Con eso proyecta."

05 Resultados y Demo ~3 min
"Los resultados superaron nuestras propias metas.

El concurso exigía un MAE menor a 1.0 punto porcentual — logramos 0.43. Un RMSE menor a 1.5 — logramos 0.54. Pasamos los 18 tests automatizados de calidad de datos y modelo. Y el intervalo de confianza es del 90%.

¿Qué significa 0.43 pp en la práctica? Si la tasa real de desempleo es 10.5%, el modelo predice entre 10.07% y 10.93%. Para un indicador macroeconómico mensual, eso es una precisión del 96%.

También encontramos hallazgos relevantes: Cúcuta tiene informalidad superior al 65%, más del doble que Bogotá con 31%. Un ANOVA confirma que esas diferencias son estadísticamente significativas entre ciudades. Y el SENA redujo sus inscritos en manufactura un 18% entre 2019 y 2020, justo cuando el desempleo alcanzó su pico pandémico.

El dashboard está publicado ahora mismo en economia-empleo.vercel.app — se los muestro."

Preguntas más probables del jurado

¿Por qué Holt-Winters y no una red neuronal?
Con 34 puntos de datos una red neuronal sobreajusta — aprende el ruido. Holt-Winters es el modelo correcto para esta cantidad de datos con tendencia y estacionalidad clara. Los resultados lo confirman: 0.43 pp de MAE.
¿Cómo garantizan que no está sobreajustado?
División estricta hold-out en el tiempo — el modelo nunca vio los últimos 7 meses durante el entrenamiento. Las métricas son exclusivamente del conjunto de prueba. Los parámetros α, β, γ se optimizan automáticamente, no los ajustamos manualmente.
¿Por qué los datos del SENA son solo de 2019–2020?
El dataset disponible en datos.gov.co con identificador 8pqf-rmzr — que cumple el requisito del concurso de estar en el portal oficial — tiene cobertura hasta 2020. Usamos ese porque es el que está disponible públicamente.
¿El dashboard se actualiza automáticamente?
La arquitectura está diseñada para eso. Los scripts de extracción pueden correr periódicamente y el modelo se reentrena con los nuevos datos. En la versión actual el proceso es manual — se ejecuta cuando el DANE publica un nuevo boletín — pero está listo para automatizarse.
¿Qué es PySpark y para qué lo usaron?
PySpark es el API de Python para Apache Spark, un motor de procesamiento distribuido. Lo usamos para procesar los 40+ Excel en paralelo y cargarlos a PostgreSQL vía JDBC. Escala sin cambios si el volumen de datos crece.
¿Qué significa el intervalo de confianza del 90%?
Que el valor real tiene un 90% de probabilidad de estar dentro de la banda que muestra el gráfico. Se calcula como la predicción más o menos 1.645 veces el MAE. Para el primer mes la banda es de aproximadamente ±0.7 puntos porcentuales.
