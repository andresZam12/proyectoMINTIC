1. ¿Qué es y qué problema resuelve?

La información del mercado laboral colombiano está dispersa en tres fuentes oficiales incompatibles: boletines PDF mensuales del DANE, archivos Excel trimestrales del FILCO/MinTrabajo y una API REST del SENA. No existe un punto único donde un tomador de decisiones pueda ver el desempleo histórico, comparar la informalidad por ciudad y conocer la proyección para los próximos meses.

Este proyecto integra las tres fuentes, construye un data warehouse en PostgreSQL y expone los resultados en un dashboard web público con un modelo de IA que pronostica la tasa de desocupación a 6 meses.

4. Implementación de Inteligencia Artificial

Modelo principal: Holt-Winters (Suavización Exponencial Triple)

Se implementa ExponentialSmoothing de statsmodels con componentes de nivel, tendencia aditiva y estacionalidad aditiva de período 12 meses. El modelo fue seleccionado por su interpretabilidad, bajo costo computacional y buen desempeño en series con estacionalidad anual marcada. 

from statsmodels.tsa.holtwinters import ExponentialSmoothing

model = ExponentialSmoothing(
    serie_train,
    trend='add',        # tendencia aditiva
    seasonal='add',     # estacionalidad aditiva
    seasonal_periods=12  # ciclo anual (datos mensuales)
).fit(optimized=True, use_brute=True)  # optimiza alpha, beta, gamma automáticamente

Análisis estadístico complementario: ANOVA de una vía

Se aplica scipy.stats.f_oneway sobre las tasas de informalidad por ciudad para determinar si existen diferencias estadísticamente significativas entre ciudades (p-valor < 0.05). El resultado confirma heterogeneidad regional y justifica el análisis desagregado por ciudad en el dashboard.

Detección automática Prophet / Holt-Winters

El pipeline intenta importar Prophet (Meta/Facebook) primero. Si no está disponible (requiere compilador C++ y Stan), cae automáticamente a Holt-Winters. En producción se usa Holt-Winters por su compatibilidad universal.

El dashboard tiene 4 vistas principales:

12. Cómo reproducir el proyecto desde cero

# 1. Clonar el repositorio
git clone https://github.com/andresZam12/proyectoMINTIC.git
cd proyectoMINTIC/proyecto_mintic

# 2. Instalar dependencias
pip install -r requirements.txt

# 3. Configurar credenciales (crear archivo .env)
# POSTGRES_HOST=localhost  POSTGRES_PORT=5432
# POSTGRES_DB=mintic_db   POSTGRES_USER=mintic_user  POSTGRES_PASSWORD=mintic2026

# 4. Levantar infraestructura Docker
docker-compose up -d

# 5. Ejecutar pipeline completo
python pipelines/01_datasource/extraccion_sena.py
python pipelines/01_datasource/extraccion_dane_boletines.py
python pipelines/02_dataprocess/parsear_boletines.py
python pipelines/03_datatransform/transformar_informalidad.py
python pipelines/pipeline_ml.py    # entrena modelo y genera forecast
Resultado esperado: el pipeline genera data/04_model_output/prediccion_td.csv con el pronóstico a 6 meses, models/holt_winters_td.pkl con el modelo serializado y reports/figures/forecast_td.png con la gráfica del forecast.
